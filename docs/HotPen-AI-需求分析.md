## 一、背景与目标

### 1.1 背景

在信息爆炸的时代，热点事件和新闻更新速度极快，内容创作者面临两大核心挑战：

1. **时效性**：热点事件发生后，需要在最短时间内产出优质内容
2. **深度**：浅层搬运已经无法满足读者需求，需要有深度、有引用、有价值的文章

### 1.2 战略目标

| 目标 | 说明 | 战略关联 |
|------|------|----------|
| **缩短内容生产周期** | 从热点发生到文章发布，从 2-4 小时缩短至 30 分钟以内 | 内容时效性决定流量获取能力 |
| **提升内容质量** | 通过多轮检索+论文引用，打造差异化深度内容 | 深度内容是留存关键 |
| **降低人工成本** | 自动化抓取→研究→撰写→发布全流程 | 提效降本的核心诉求 |

### 1.3 成功指标

| 指标 | 当前值 | 目标值 | 衡量周期 |
|------|--------|--------|----------|
| 热点发现到内容产出耗时 | 2-4 小时 | ≤30 分钟 | 每次热点 |
| 内容采用率（审核通过率） | - | ≥70% | 每周 |
| 单篇内容平均阅读量 | - | ≥5000 | 每月 |
| 人工干预时间/篇 | 30 分钟 | ≤10 分钟 | 每周 |

## 二、用户与场景

### 2.1 目标用户

| 用户群 | 描述 |
|--------|------|
| **自媒体创作者** | 需要持续产出热点内容的个人或团队 |
| **内容工作室** | 有内容生产需求的小型团队 |
| **企业新媒体** | 需要快速响应热点的企业运营团队 |

### 2.2 核心场景

**场景 1：科技热点快速产出**
- 作为 **自媒体创作者**
- 我希望 **热点发生后快速产出深度科技文章**
- 以便 **第一时间获得流量先发优势**

**场景 2：学术引用增强权威**
- 作为 **内容创作者**
- 我希望 **文章能引用权威论文**
- 以便 **提升内容可信度和读者信任**

## 四、Agents 设计

### 4.1 热点抓取 Agents (5个)

| Agent | 领域 | 数据源 |
|-------|------|--------|
| HotTracker-Tech | 科技 | 36氪、TechCrunch、虎嗅 |
| HotTracker-Finance | 财经 | 财经网、同花顺、雪球 |
| HotTracker-Entertainment | 娱乐 | 微博热搜、豆瓣、知乎热榜 |
| HotTracker-Social | 社会 | 人民日报、澎湃新闻 |
| HotTracker-AI | AI | 机器之心、AI科技评论 |

### 4.2 核心 Agents

| Agent | 职能 |
|-------|------|
| **Aggregator** | 热点聚合、去重、评分 |
| **Researcher** | 深度研究、学术论文检索 |
| **Writer** | 内容生成（公众号/小红书）|
| **Publisher** | 多平台发布 |
| **Scheduler** | 任务调度与编排 |
| **Backend** | 后端代码开发 |
| **Frontend** | 前端代码开发 |
| **TestAgent** | 测试代码开发 |
| **QAgent** | 代码质检 |
